# LoopGuard CTX Vision

`loopguard-ctx` exists to make AI coding sessions smaller, cleaner, and less wasteful.

## The job

When an AI tool reads too much, it gets slower, noisier, and more expensive.

The helper keeps the next turn focused by doing less, not more:

- smaller reads instead of full-file dumps
- cleaner shell output instead of raw logs
- optional session restore instead of repeating the same background every time

## What good looks like

- A model sees the failing window, the nearby definitions, and the recent edits.
- A terminal command returns the lines that matter, not five screens of noise.
- A returning agent can pick up where it left off without a full cold start.

## What this is not

- not a cloud service
- not a surveillance layer
- not a research demo
- not a promise that every workflow will hit the same compression percentage

## Product principle

Selection beats bulk.

The point of the helper is not to sound clever. The point is to reduce how much irrelevant context reaches the model so developers spend less time and less money getting to the next useful answer.
