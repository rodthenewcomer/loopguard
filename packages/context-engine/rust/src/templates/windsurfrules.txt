# LoopGuard tools

Prefer LoopGuard MCP tools when you need smaller reads and cleaner command output.

## Use these mappings

- Read -> ctx_read
- Grep / rg -> ctx_search
- terminal commands with long output -> ctx_shell
- ls / find -> ctx_tree

## Read modes

- full for edits
- map for structure
- signatures for API surface
- diff after changes
- lines:N-M for exact ranges

## Session restore

If you are coming back to a previous helper session:

  ctx_session load

## Project knowledge

Store and retrieve facts any agent can access:

  ctx_knowledge(action="set", key="arch.db", value="Supabase Postgres with RLS", category="architecture")
  ctx_knowledge(action="get", key="arch.db")
  ctx_knowledge(action="list")

## Multi-agent handoff

Hand off work to another agent or pick up where one left off:

  ctx_agent(action="write", agent="windsurf", label="current-task", content="...")
  ctx_agent(action="read",  label="current-task")
  ctx_agent(action="list")
